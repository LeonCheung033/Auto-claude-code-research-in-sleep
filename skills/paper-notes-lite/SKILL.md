# Paper Notes Lite - 论文深度阅读与结构化分析

将学术论文（PDF 或 arXiv 链接）整理为结构化的详细阅读笔记（Markdown）。
专注论文内容理解，无外部依赖，适合作为 auto research pipeline 的精读模块。

## 触发关键词

论文笔记、paper notes、读论文、论文阅读、阅读笔记、整理论文、paper reading、arXiv、论文总结

## 执行流程

### Step 1: 获取论文内容

**情况 A — PDF 附件**
1. 使用 PDF 提取工具获取全文（分批提取，每次最多 50 页）
2. 将提取文本保存到工作区文件

**情况 B — arXiv 链接**
1. 从链接中提取 arXiv ID
2. 构造 PDF 链接 `https://arxiv.org/pdf/{arxiv_id}.pdf` 并提取全文
3. 提取失败时，访问 abs 页面获取摘要信息作为补充

### Step 2: 提取表格与伪代码

**表格提取（必做）**：
1. 扫描全文中所有 `Table X` 标记，记录标题
2. 重建表格结构，转换为 Markdown 表格
3. 每张表格上方添加：`> 论文 Table X — [caption]`
4. 每张表格下方添加 1-2 条关键发现

**图片处理（文字描述）**：
- 对所有 Figure，基于正文中的引用和描述，用文字重建图片内容
- 格式：`> **[Figure X — 标题]**: [图片类型、主要模块、流向、关键标注、数据趋势]`

**伪代码提取**：
- 将 Algorithm 伪代码转换为 fenced code block

### Step 3: 全面阅读与分析

重点关注：
- Title, Authors, Affiliations, Venue, Date
- Abstract / Introduction：问题定义与动机
- Related Work：技术路线梳理
- Method：每个模块、公式、算法
- Experiments：所有表格、数字、消融实验
- Conclusion / Limitations
- 代码 / 数据集 / 项目链接

### Step 4: 按模板组织笔记

严格按照以下六大部分结构输出，**不遗漏任何子章节**：

---

#### 笔记模板

````
# [论文标题]
> [作者列表] | [机构] | [发表渠道] | [发表日期]
> 阅读日期：[YYYY-MM-DD]

---

## 第0部分：速览 (At a Glance)

### 0.1 一句话总结
用一句话概括：这篇论文用 **[什么方法]** 解决了 **[什么领域]** 中 **[什么问题]**。

### 0.2 核心四问
| 问题 | 回答 |
|------|------|
| **要解决什么问题?** | [1-2 句话精确描述] |
| **用了什么方法?** | [核心技术路线概述] |
| **效果怎么样?** | [关键量化结果，包含具体数字和 benchmark] |
| **对我的启发?** | [对个人研究/工作最有价值的 takeaway] |

### 0.3 元信息
- **关键词**: [5-8 个关键词]
- **贡献类型**: [✅ 新方法 / ✅ 新理论 / ✅ 综合实验 / ✅ 新数据集 / ✅ 综述]
- **源码**: [链接或"未公开"]
- **数据集/模型**: [使用或构建的关键资源]
- **项目主页**: [链接或"无"]

## 第1部分：Situation（背景 — 领域地图）
> 业界目前在做什么？这篇文章是在什么大背景下提出的？

**⚠️ 本部分只回答"有什么"，严禁出现批评、不足、局限的措辞。负面评价留给第2部分。**

### 1.1 问题的重要性
2-3 句话说明该研究问题为什么重要，交代领域大背景。

### 1.2 主流技术路线扫描
梳理 2-3 条主流技术路线，每条包括：
- **路线名称**：核心思想一句话
- **代表工作**：1-2 篇有影响力的论文
- **适用场景**：在什么条件下表现好

## 第2部分：Motivation（动机 — 从 Gap 到问题定义）
> 现有路线在什么场景下失效？作者如何将 gap 转化为可研究的问题？

**⚠️ 聚焦"缺什么 → 为什么缺 → 怎么定义"，回指第1部分的路线编号。**

### 2.1 Gap 识别：现有路线的失效场景
逐条指出 3-4 个失效场景：
- **Gap N — [短标签]**（对应路线 X）：[失效场景和现象]

### 2.2 核心 Insight：作者的关键发现
- **核心观察**：[gap 背后的共同根源]

### 2.3 问题定义
- **任务形式化**：[数学公式用 $$ 包裹]
- **与已有任务的区别**：[一句话]
````

## 写作规范

### 语言风格
- **中文为主**，专业术语保留英文原文
- **加粗** 标出关键概念、方法名称、重要结论
- 表格用于结构化对比信息
- 内容要足够**详细和深入**，每个部分都应有实质性内容

### 数学公式规范
- **所有公式使用双美元符号 `$$` 包裹**
- 行内：`$$E = mc^2$$`
- 独立公式块：`$$\mathcal{L} = -\sum_{i=1}^{N} y_i \log(\hat{y}_i)$$`
- **尖括号转义**：`<` → `\lt`，`>` → `\gt`，防止被解析为 HTML 标签
- **禁止**单美元符号 `$...$`、`\[...\]`、`\(...\)` 格式

### 表格规范
- 论文中每张 Table 必须转为 Markdown 表格，不允许遗漏
- 数字与原文完全一致，最优结果 **加粗**
- 主实验表 → 第4部分对应实验小节；所有表格完整版 → 附录

### 图片处理（纯文字描述）
- 所有 Figure 用文字详细描述，不拉取图片
- 格式：`> **[Figure X — 标题]**: [图片类型、主要模块、箭头流向、关键标注、数据趋势等]`
- 基于论文正文中对 Figure 的引用和描述来重建内容

### "我的思考"写作要求
- **资深 ML 研究者/工程师**视角
- **批判性分析**，不只说好话
- 具体到方法细节，避免空泛评价

### 内容详尽度
- 方法部分：每个模块展开，含公式和算法流程
- 实验部分：包含具体数字，表格以 Markdown 嵌入
- 消融实验：逐条列出
- 总体字数 **3000-8000 字**

## 注意事项

1. **不遗漏任何子章节**：模板中每个编号章节都必须出现
2. **优先从论文原文提取信息**：不臆造数据或实验结果
3. **信息缺失时**：标注"论文未详细说明"而非跳过
4. **公式中 `<` / `>`**：用 `\lt` / `\gt` 替代，防止 HTML 解析截断
5. **表格数字准确性**：与原文完全一致
6. **Algorithm 伪代码**：转为 fenced code block
